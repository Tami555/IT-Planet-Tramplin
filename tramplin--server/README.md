# Трамплин — Карьерная платформа

> Платформа для централизованного взаимодействия студентов, выпускников, работодателей и карьерных центров вузов в сфере IT.

---

## Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| Фреймворк | **NestJS** | Модульная архитектура, встроенный DI, TypeScript из коробки, отличная интеграция со Swagger |
| ORM | **Prisma** | Типобезопасные запросы, авто-генерация клиента, удобные миграции |
| БД | **PostgreSQL** | Надёжная реляционная СУБД, поддержка массивов (для навыков, тегов), полнотекстовый поиск |
| Аутентификация | **JWT (Access + Refresh)** | Stateless-авторизация, ротация токенов для безопасности |
| Документация | **Swagger (OpenAPI 3.0)** | Интерактивная документация прямо в проекте |
| Файлы | **Multer + локальное хранилище** | Простая интеграция, легко заменить на S3 |
| Валидация | **class-validator + class-transformer** | Декларативная валидация DTO |
| Rate Limiting | **@nestjs/throttler** | Защита от брутфорса и DDoS |

## Быстрый старт

### 1. Клонировать и установить зависимости

```bash
git clone <repo>
cd tramplin
npm install
```

### 2. Настроить переменные окружения

```bash
cp .env.example .env
# Отредактируйте .env — укажите DATABASE_URL и JWT-секреты
```

### 3. Применить миграции и заполнить начальные данные

```bash
npm run prisma:migrate   # Создаёт таблицы
npm run prisma:generate  # Генерирует Prisma Client
npm run prisma:seed      # Создаёт администратора + системные теги
```

### 4. Запустить сервер

```bash
npm run start:dev   # Режим разработки (hot-reload)
npm run start:prod  # Production
```

### 5. Открыть документацию

```
http://localhost:3000/api/docs
```

## Авторские решения (зоны неопределённости ТЗ)

### 1. Верификация работодателей — двухуровневая система

**Реализация:** При подаче заявки работодатель предоставляет один или оба идентификатора:
- **Корпоративный email** — сервис автоматически проверяет соответствие домена email сайту компании
- **ИНН** — куратор вручную проверяет по открытым данным ЕГРЮЛ

**Обоснование:** Комбинация автоматической и ручной проверки даёт баланс между удобством и надёжностью. Автоматическая проверка email снижает нагрузку на куратора для крупных компаний, ИНН — надёжный идентификатор для малого бизнеса и ИП.

### 2. Система модерации — очередь с комментариями

**Реализация:** После создания/редактирования вакансия получает статус `MODERATION` и попадает в очередь куратора. Куратор одобряет или отклоняет с комментарием (`moderationNote`), который видит работодатель.

**Обоснование:** Асинхронная модерация не блокирует работу платформы. Комментарий при отклонении помогает работодателю исправить вакансию, а не создавать дублирующий запрос.

### 3. Приватность — трёхуровневая система (PRIVATE / CONTACTS / PUBLIC)

**Реализация:** Соискатель независимо настраивает видимость профиля, резюме и откликов. При уровне `CONTACTS` проверяется наличие принятой связи в таблице `contacts`.

**Обоснование:** Три уровня покрывают все сценарии: закрытый профиль для активных джуниоров (боятся засветиться у текущего работодателя), контактный — для нетворкинга внутри платформы, публичный — для активного поиска.

### 4. Карточка для карты — модульный состав

**Состав:** название, компания, логотип, тип, зарплата, первые 3 тега, координаты.

**Обоснование:** Минимальный набор для быстрой оценки вакансии прямо с карты без перехода на полную карточку.

### 5. Профессиональные контакты — двустороннее подтверждение

**Реализация:** Запрос → ожидание → принять/отклонить. Контакты видят карьерные интересы и навыки друг друга согласно настройкам приватности.

### 6. Работодатель видит соискателей только по откликам

**Обоснование:** Защита от несанкционированного просмотра профилей. Работодатель не может «охотиться» на пассивных соискателей — только те, кто явно отозвался.

## Безопасность

- **Пароли** — bcrypt с salt rounds = 12
- **JWT** — разные секреты для access и refresh токенов
- **Ротация токенов** — refresh-токен инвалидируется при каждом использовании
- **Rate limiting** — 100 запросов / 60 секунд на IP
- **Валидация** — whitelist + forbidNonWhitelisted (лишние поля отклоняются)
- **RBAC** — ролевой контроль доступа через `@Roles()` + `RolesGuard`
- **Изоляция данных** — работодатель видит только свои вакансии и отклики
